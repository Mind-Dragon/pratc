# prATC v1.7 WS4 — Contract Widening Plan

> For Hermes: implement this in small commits, with a failing contract test before each behavior change. Keep `main` green after every merge.

Goal: finish WS4 by widening the right service/client contracts instead of faking fixes at the HTTP layer.

Architecture:
- Add a real `PlanOptions` path from HTTP + CLI into `app.Service.Plan`
- Make review failure semantics explicit instead of accidental
- Unify token fallback behavior across GraphQL and REST client paths

Tech stack: Go, stdlib, existing `internal/app`, `internal/cmd`, `internal/github`, `internal/settings`, existing test suite.

---

## Why WS4 is blocked today

Current blockers are structural, not local:
1. `handlePlan()` parses params the service cannot accept.
2. `Analyze()` / `buildReviewPayload()` currently choose partial-success semantics implicitly.
3. GraphQL and REST fallback do not share one token-source / retry contract.

This plan removes those blockers in 3 subtracks.

---

# Track A — PlanOptions contract widening

## Target end state

Replace the narrow call shape:
- `Plan(ctx, repo, target, mode)`

With an additive shape:
- `Plan(ctx, repo, PlanOptions)`

Where `PlanOptions` includes:
- `Target int`
- `Mode formula.Mode`
- `ExcludeConflicts bool`
- `CandidatePoolCap int`
- `ScoreMin float64`
- `StaleScoreThreshold float64`
- `CollapseDuplicates bool`
- `DryRun bool`

Do not delete the old method until all call sites are migrated. If needed, keep a compatibility shim for one pass.

# Track B — Review failure semantics

## Decision required before implementation

Pick one of these and encode it explicitly:

Option 1 — Fail-fast
- any review pipeline failure causes `Analyze()` to return error
- simplest semantics
- strictest API

Option 2 — Partial-success, explicit degradation
- `Analyze()` returns partial analysis with explicit degradation markers
- requires response schema additions
- better operator continuity, more complex contract

Recommended: Option 2 if prATC is intended to remain usable under partial backend degradation.
