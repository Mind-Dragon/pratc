## Failure Path: Empty MINIMAX_API_KEY

Command: MINIMAX_API_KEY="" docker compose --profile minimax-light up -d --wait --wait-timeout 60
Result: SUCCESS (containers started)

### Behavior Analysis

The minimax-light Docker Compose profile does NOT pass MINIMAX_API_KEY to the Go API service.
The docker-compose.yml only sets PRATC_ANALYSIS_BACKEND (defaults to "local").
Minimax is an ML-service backend, not a Go-backend concern at the container level.

The Go API service starts successfully regardless of MINIMAX_API_KEY because:
1. PRATC_ANALYSIS_BACKEND defaults to "local" (not "minimax") in compose config
2. Minimax is configured via ML_BACKEND env var only when ml-service is invoked
3. The API service itself does not require MINIMAX_API_KEY to start
