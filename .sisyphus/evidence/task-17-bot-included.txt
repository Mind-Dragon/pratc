### Flag Wiring Verification
The --include-bots flag is correctly wired through the call stack:
1. CLI flag parsed in RegisterPlanCommand() -> includeBots bool
2. Passed to app.NewService(app.Config{IncludeBots: includeBots})
3. Service.Plan() passes includeBots to planner.New(planner.WithIncludeBots(s.includeBots))
4. Planner passes includeBots to filter pipeline via WithIncludeBots()
5. Filter.ApplyFilters(prs, includeBots) accepts bots when includeBots=true

## Implementation Details

When --include-bots is provided:
- Config.IncludeBots = true
- planner.WithIncludeBots(true) is called
- Pipeline is created with includeBots=true
- ApplyFilters(prs, true) skips the bot filter
- Bot PRs remain in candidate pool and are eligible for selection

When --include-bots is NOT provided (default):
- Config.IncludeBots = false  
- planner.WithIncludeBots(false) is called (default)
- Pipeline is created with includeBots=false
- ApplyFilters(prs, false) rejects PRs where IsBot=true
- Bot PRs are excluded with rejection reason "bot pr"
