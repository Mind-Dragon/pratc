Bot Detection Patterns (v0.1 hardcoded):
- Authors: dependabot[bot], renovate[bot], github-actions[bot], snyk-bot
- Titles: ^Bump , ^chore\(deps\), ^Update dependency

Bot PRs are clustered into "dependency updates" group via existing clusterKey() logic.
By default, bots are EXCLUDED from merge plans (is_bot filter applied).

Note: The fixture dataset (fixtures/prs/) does not contain any bot PRs, so the
default plan command shows 0 "bot pr" rejections. The functionality is proven
correct by the unit tests which explicitly test bot PR detection and filtering.
