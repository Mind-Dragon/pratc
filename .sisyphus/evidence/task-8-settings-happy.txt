Task 8: Settings API Export/Type Gaps - Happy Path Evidence
=============================================================

Date: 2026-03-22

## Summary
Fixed settings API exports and type gaps so settings page compiles cleanly.

## Changes Made
1. Added settings functions to `web/src/lib/api.ts`:
   - `fetchSettings(repo)` - fetches settings map from API
   - `postSetting(payload, options)` - posts setting with optional validation
   - `deleteSetting(scope, repo, key)` - deletes a setting
   - `exportSettingsYAML(scope, repo)` - exports settings as YAML
   - `importSettingsYAML(scope, repo, content)` - imports settings from YAML

2. Added settings types to `web/src/lib/api.ts`:
   - `SettingsMap` - key-value record type for settings
   - `PostSettingPayload` - payload structure for posting settings
   - `PostSettingOptions` - options including validateOnly
   - `PostSettingResponse` - response with updated/valid flags
   - `DeleteSettingResponse` - response with deleted flag
   - `ImportSettingsResponse` - response with imported flag

3. Moved test file from `web/src/pages/settings.test.tsx` to 
   `web/src/__tests__/pages/settings.test.tsx` to fix Next.js build issue
   (test files in pages directory were being picked up as pages).

4. Added comprehensive API tests in `web/src/__tests__/lib/api.test.ts`
   covering all settings functions with success and error scenarios.
