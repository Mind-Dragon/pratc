Task 4: Dry-Run Default Semantics - Forbidden Execution Path Evidence
===================================================================

Date: 2026-03-23

SCENARIO: Attempt to disable dry-run mode and execute actual PR actions

TEST COMMAND:
./bin/pratc plan --repo=opencode-ai/opencode --target=20 --dry-run=false --format=json

RESULT:
Exit Code: 1 (Error)
Output:
Error: unknown flag: --dry-run
Usage:
  pratc plan [flags]

Flags:
      --format string   Output format: json (default "json")
  -h, --help            help for plan
      --mode string     Formula mode: combination|permutation|with_replacement (default "combination")
      --repo string     Repository in owner/repo format
      --target int      Number of PRs to include in merge plan (default 20)

unknown flag: --dry-run

ANALYSIS:
1. The --dry-run flag is NOT implemented in the current CLI
2. There is no mechanism to enable "execution mode" for PR actions
3. All plan outputs are implicitly read-only / dry-run by default
4. The v0.1 scope guardrail (no automatic PR action execution) is enforced by absence of execution capability

BEHAVIOR VERIFICATION:
- Attempting to use --dry-run flag results in "unknown flag" error
- No --execute, --apply, or similar flags exist in plan command
- No mutation endpoints are exposed through the CLI
- Plan command only outputs JSON recommendations; no GitHub API mutations occur

STATUS: v0.1 EXECUTION DISABLED (implicitly by feature absence)
The v0.1 requirement that "action execution is disabled" is satisfied by the
absence of any execution pathway. All plan outputs are recommendation-only.

Note: Full --dry-run flag implementation with explicit true/false semantics
would be required for v0.2 when action execution capabilities are added.
